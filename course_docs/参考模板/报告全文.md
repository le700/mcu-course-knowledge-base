综合设计任务
============

P-A-3\# 板：温度记录器设计

PC 机与下位单片机进行串行通讯， 单片机采用串行通讯方式 1，波特率
4800。PC 机采用"串口助手"与单片机进行通讯， 在 PC
机"串口助手"的界面中将波特率设置成 4800，串口选择右边的 USB 转串口的
COM，打开串口。先将单片机目标板的电位器调到最大。

第一步：在 PC 机"串口助手"发送缓冲区内输入任意数，
点击"发送数据"按钮；下位单片机目标板接收该数据，并在 8
个发光二极管（L1-L8）中显示该数据； 同时单片机将该数据串行送回 PC 机，在
PC机"串口助手"接收缓冲区内回显发送的数据；当接收到特定数据（自定义）时，方可进行下一步。

第二步：当按键 S4 按下后， 8 个发光二极管（L1-L8）全亮，进行下一步。

第三步： 当按下 S5 时，通过 PCF8591 读取电压信号（RB2），假设 0-5V
的模拟信号对应 0-80 度的温

度，要求在数码管中显示出相应的温度（含两位小数点），调节电位器观察数值变化情况。显示要求如下：

+-----+---+---+------+-----+-------+-----+-----+
| > A |   |   | > \| | > 2 | > 3\. | > 4 | > 5 |
+-----+---+---+------+-----+-------+-----+-----+

第四步： 当按下 S6 时，通过 DS18B20 读取真实的温度信息，
在数码管中显示出相应的温度（含两位

小数点）。显示要求如下：

+-----+---+---+------+-----+-------+-----+-----+
| > B |   |   | > \| | > 2 | > 7\. | > 1 | > 1 |
+-----+---+---+------+-----+-------+-----+-----+

else SDA = 0; // 最高位为0则SDA置低

SCL = 0; I2C\_Delay(); // 时钟置低，允许数据变化

3.调试中遇到的问题及解决方法
----------------------------

问题1：串口通信阶段输入特定数据AA后，无对应LED灯显示，且串口功能未关闭

解决方法：首先在串口处理函数Proc\_UART()中添加数据判断分支，当检测到接收数据rx\_data
== 0xAA时，将系统状态sys\_state置1（预锁定状态），同时执行ES =
0关闭串口中断，彻底禁用串口接收功能；其次为普通数据和特定数据分别添加LED灯控逻辑，普通数据接收时将数据取反后通过P0口输出至LED通道，实现数据的LED可视化显示，特定数据AA接收时触发LED闪烁提示，完成状态跳转的硬件反馈；最后在主循环中增加状态判断，确保系统进入预锁定和终态锁定后，串口处理函数不再响应任何数据，形成"软件逻辑+硬件动作+状态锁定"的完整串口通信流程。

问题2：数码管显示重影、闪烁、数字模糊

解决方法：每次切换位选前先关闭段选，把 P0 置全灭 0xFF
做消影。调整定时器中断为
2ms扫描一次，保证视觉暂留稳定并严格遵循：位选→段选→延时→关闭 的流程。

问题3：PCF8591 ADC 采集不准，电压转温度偏差大

解决方法：修正 I2C
起始、停止、读写时序，保证足够微秒级延时。固定采集通道
3（RB2），发送控制字 0x43。使用公式：温度 = (ADC值 × 8000) / 255
保证两位小数准确。

问题4：按键 S4、S5、S6 按下无反应

解决方法： 加入 20ms 软件消抖，确认按键按下有效再执行逻辑。
按步骤判断：串口解锁→S4 进入运行→S5/S6 切换模式。
检查引脚定义：S4=P3.3、S5=P3.2、S6=P3.1 是否正确。

在这一次单片机实训的课程中，我通过绘制硬件原理接线图，对于构成单片机的硬件有了全面且清晰的了解。同时我也认识到绘制仿真原理图与实际单片机之间是有区别的，在学习的过程中应当对照原理图与实物图，这样才能帮助我们更好的来理解单片机，避免只精通理论使学习成为空中楼阁的问题。这一次实验的开始是困难的，因为我们以往并没有面对单片机的经验，需要从头开始学习软件编程以及硬件调试。起初我们可能会因为一个很小的问题而卡住导致进度停止，但我们经过小组之间的积极沟通和在网上进行大量相关知识的调研，成功依靠自己的力量攻克一个又一个难关其中的成就感是难以言喻的。

学生C：

本次单片机工程实训非常有收获。在这个实训中，我们使用了51系列单片机，并学习了单片机的基础知识和编程技巧。我对单片机有了更深入的了解。在此之前，我只是简单地了解了一些单片机的基本概念，如引脚、寄存器、时钟等。但通过这次实训，我深入了解了它们的内部原理，比如8051架构、指令集和存储器映射等。这使我能够更好地理解单片机的工作原理，从而更好地设计程序。同时作为组长不仅要自己理解单片机，还要能够指导组员合理分配任务让每个人都能在单片机实训中有所收获，成功完成老师安排给我们的任务。
