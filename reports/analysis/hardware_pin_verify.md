### 1.1 段码位映射验证

**硬件映射：** P0.0=bit0=a, P0.1=bit1=b, P0.2=bit2=c, P0.3=bit3=d, P0.4=bit4=e, P0.5=bit5=f, P0.6=bit6=g, P0.7=bit7=dp

**判定：** ✅ **位映射正确。** 这是标准 51 单片机共阴数码管段码映射，与标准段码表一致：
- 数字 '0' = 0x3F (a,b,c,d,e,f 亮)
- 数字 '1' = 0x06 (b,c 亮)
- '-' = 0x40 (仅 g 段亮)
- 'E' = 0x79 (a,d,e,f,g 亮)

**注意：** 由于源代码 (main.c) 不在当前工作区，无法逐行验证 `UpdateDisplay()` 中的段码表常量。代码审查报告确认函数位于 main.c 第 686-740 行，编译 0 Error 0 Warning，逻辑推断正确。

### 1.2 共阴数码管段码逻辑验证

**判定：** ✅ **逻辑正确。** 共阴数码管：段码位=1 时该段亮。P0 输出高电平 (经上拉) → 段 LED 阳极得电 → 共阴端 (位选 P2.x=0) 灌电流 → 段亮。

### 1.3 P0 口驱动能力分析（关键问题）

**问题描述：**

P0 口为开漏输出，无内部上拉。系统使用 8×10kΩ 外部上拉电阻 (Bug #2 已修复)。数码管段驱动路径为：

```
VCC → 10kΩ(上拉) → P0.x 引脚 → 220Ω(限流) → 段 LED → 共阴端 → P2.x(位选,灌电流) → GND
```

**电流计算：**
- LED 正向压降 V_F ≈ 1.8-2.2V (红色)
- 每段电流 I_SEG ≈ (5V - 1.8V) / (10kΩ + 220Ω) ≈ 3.2V / 10.22kΩ ≈ **0.31mA**

**对比需求：**
- 标准 5mm LED 数码管段：需要 **5-10mA** 才能正常亮度
- 当前设计提供：**0.31mA**，约为需求的 1/20 到 1/30

**判定：** ❌ **P0 口驱动能力严重不足，10kΩ 上拉电阻过大。**

| 参数 | 设计值 | 推荐值 | 差距 |
|:---|:---:|:---:|:---:|
| 上拉电阻 | 10kΩ | 470Ω-1kΩ | 10-20倍 |
| 每段电流 | ~0.31mA | 5-10mA | 16-32倍 |
| 4位全亮总电流 | ~1.24mA | 40-80mA | 32-64倍 |

**影响分析：**
- 在 Proteus 仿真中：数码管模型可能以极低电流显示为"亮"，仿真通过但实际不可用
- 在实际硬件中：数码管段极度昏暗，几乎不可见
- 220Ω 限流电阻在 10kΩ 上拉下形同虚设 (10kΩ 是瓶颈)

**修正建议：**

**方案 A（推荐）：将上拉电阻改为 470Ω×8**
- 每段电流：I ≈ (5V - 1.8V) / (470Ω + 220Ω) ≈ 3.2V / 690Ω ≈ **4.6mA**
- 4 位全亮总电流：4 × 8 × 4.6mA = 147mA (需确认电源能力)
- 动态扫描占空比 1/4，平均电流 ~1.15mA/段，亮度可接受
- 上拉电阻功耗：P = (5V)²/470Ω × 8 = 53mW×8 = 425mW 总功耗 (可接受)

**方案 B：增加 74HC245 总线驱动**
- P0 口 → 74HC245 → 220Ω → 数码管段
- 74HC245 输出驱动能力：±35mA
- 保留 10kΩ 上拉 (仅用于 P0 口本身电平)
- 这是工业标准做法，可靠性和驱动能力最佳

**方案 C（仅适用于 Proteus 仿真）：保持 10kΩ 上拉**
- Proteus 仿真中数码管模型可能正常工作
- 标注"仿真专用，实际硬件需改小上拉电阻或增加驱动芯片"

### 2.1 LED 驱动方式验证（关键问题）

**硬件设计文档描述：** "写1点亮" — P1.x=1 时 LED 亮。

**电路结构：** P1.x → 220Ω → LED 阳极 → LED 阴极 → GND（拉电流模式）

**问题分析：**

P1 口为准双向口，内部结构如下：
- 输出高电平 (写 1)：内部弱上拉 (~50kΩ 等效)，仅能提供 **50-100μA** 拉电流
- 输出低电平 (写 0)：内部 MOSFET 导通，可灌入 **10-15mA** 电流

**LED 电流计算 (拉电流模式，当前设计)：**
- I_LED ≈ (5V - V_F) / (50kΩ + 220Ω) ≈ 3.2V / 50.22kΩ ≈ **64μA**
- 5mm LED 正常亮度需要 5-20mA
- 64μA 仅为需求的 **1/80 到 1/300**

**判定：** ❌ **P1 口准双向口拉电流驱动 LED 的设计不可行。** 64μA 的电流无法点亮 LED。

**修正建议：**

**方案 A（推荐，改为灌电流模式）：**
```
VCC → LED 阳极 → LED 阴极 → 220Ω → P1.x
P1.x = 0 → LED 亮 (灌电流)
P1.x = 1 → LED 灭
```
- 灌电流：I_LED ≈ (5V - V_F - V_OL) / 220Ω ≈ (5V - 1.8V - 0.3V) / 220Ω ≈ **13.2mA**
- P1 口灌电流能力 10-15mA，刚好满足
- **需要修改代码：** 将所有 LED 输出逻辑取反 (写 0 点亮，写 1 熄灭)
- **需要修改 `ApplyLights()` 函数**

**方案 B（增加 PNP 驱动）：**
```
P1.x → 1kΩ → PNP 基极 (S8550)
PNP 发射极 → VCC
PNP 集电极 → 220Ω → LED → GND
P1.x = 0 → PNP 导通 → LED 亮
```
- 驱动能力充足，但增加 6 个 PNP 三极管和 6 个基极电阻
- 电路复杂度增加

**方案 C（仅 Proteus 仿真）：**
- Proteus 中 8051 端口模型可能理想化，拉电流模式下 LED 在仿真中可能显示为亮
- 标注"仿真专用，实际硬件必须改为灌电流模式"

### 2.2 I2C 上拉电阻验证

**问题：** I2C 总线 (P1.6/SCL, P1.7/SDA) 需要外部上拉电阻。

**硬件设计文档元器件清单中无 4.7kΩ 电阻。**

**判定：** ❌ **I2C 总线缺少外部上拉电阻。**

| 项目 | 需求 | 当前状态 | 风险 |
|:---|:---|:---|:---|
| SCL 上拉 | 4.7kΩ → VCC | 仅 P1.6 内部弱上拉 (~50kΩ) | 时钟上升沿过缓，I2C 时序错误 |
| SDA 上拉 | 4.7kΩ → VCC | 仅 P1.7 内部弱上拉 (~50kΩ) | 数据线无法可靠拉高，ACK 检测失败 |

**影响分析：**
- I2C 标准模式 (100kHz)：上升时间要求 < 1000ns
- 50kΩ 上拉 + 总线电容 (~10pF)：τ = 50kΩ × 10pF = 500ns，上升时间 ~2.2τ = 1.1μs，超标
- 在 Proteus 仿真中可能出现间歇性通信失败 (Bug #10 相关)
- 软件模拟 I2C 使用 4 个 NOP 延时，时序已偏慢，加上无上拉电阻的慢上升沿，通信可靠性进一步降低

**修正建议：**
- 在 P1.6 和 P1.7 各接一个 **4.7kΩ 上拉电阻到 VCC**
- 更新元器件清单：增加 4.7kΩ 电阻 ×2

### 2.3 LED 限流电阻验证

**判定：** ✅ **220Ω 限流电阻值正确** (在改为灌电流模式后)。

灌电流模式下：I = (5V - 1.8V - 0.3V) / 220Ω ≈ 13.2mA，在 LED 额定范围内。
- 红色 LED (V_F ≈ 1.8V)：I ≈ 13.2mA ✅
- 黄色 LED (V_F ≈ 2.0V)：I ≈ 12.3mA ✅
- 绿色 LED (V_F ≈ 2.2V)：I ≈ 11.4mA ✅

**消隐逻辑确认 (Bug #1)：**
- 流程图 (图四 Timer0ISR) 确认：先关闭所有位选 (P2.0-P2.3=1)，再清 P0 段码，再送新段码，最后开位选
- 判定：✅ **消隐逻辑正确，重影问题已解决**

**位选驱动能力分析：**

P2 口为准双向口，输出低电平时灌电流能力约 **10-15mA**。

位选信号为低电平有效 (P2.x=0 选通)，共阴端需要灌入该位所有点亮段的电流：

| 场景 | 点亮段数 | 每段电流 (10kΩ 上拉) | 总灌电流 | P2 承受能力 | 判定 |

| 显示 '8' (当前设计) | 7 | 0.31mA | 2.2mA | 10-15mA | ✅ 但极暗 |
| 显示 '8' (改 470Ω 上拉) | 7 | 4.6mA | 32.2mA | 10-15mA | ❌ 超限 |
| 显示 '1' (当前设计) | 2 | 0.31mA | 0.62mA | 10-15mA | ✅ |
| 显示 '1' (改 470Ω 上拉) | 2 | 4.6mA | 9.2mA | 10-15mA | ✅ 接近上限 |

**判定：** ⚠️ **位选驱动能力取决于 P0 上拉电阻方案。**

| 情景 | P0 上拉 | 位选灌电流 | 是否可行 |
|:---|:---|:---:|:---:|
| 当前设计 | 10kΩ | 最大 2.2mA | ✅ 但数码管极暗 |
| 方案 A | 470Ω | 最大 32.2mA | ❌ 超出 P2 灌电流能力 |
| 方案 B | 加 74HC245 | 取决于 74HC245 | ✅ 需重新评估 |

**结论：** 如果采用方案 A (改 470Ω 上拉)，必须同时增加位选驱动电路。

**修正建议：**

**方案 A：增加 ULN2003A 位选驱动 (推荐)**
```
P2.0 → ULN2003A 1B → 1C → NS 十位共阴端
P2.1 → ULN2003A 2B → 2C → NS 个位共阴端
P2.2 → ULN2003A 3B → 3C → EW 十位共阴端
P2.3 → ULN2003A 4B → 4C → EW 个位共阴端
```
- ULN2003A 单通道灌电流能力：500mA
- 逻辑：P2.x=1 → ULN2003A 输出=0 (灌电流) → 位选通
- 注意：使用 ULN2003A 后位选逻辑反转 (P2.x=1 选通，而非 0 选通)
- **需要修改代码**：Timer0ISR 和消隐逻辑中的位选电平

**方案 B：增加 PNP 三极管位选驱动 (S8550×4)**
```
P2.x → 1kΩ → PNP 基极 (S8550)
PNP 发射极 → VCC
PNP 集电极 → 共阴端
P2.x = 0 → PNP 导通 → 共阴端得电 → 位选通
```
- 逻辑不变 (P2.x=0 仍为选通)
- 每个 PNP 需要 1 个基极电阻
- 增加 4 个 S8550 + 4 个 1kΩ 电阻

**方案 C（仅 Proteus 仿真）：保持现有设计**
- 配合 10kΩ P0 上拉，位选灌电流仅 2.2mA，P2 口可承受
- 标注"仿真专用，实际硬件需增加位选驱动"

### 3.2 蜂鸣器驱动验证

**电路分析：**
```
P2.7 → 500Ω → NPN 8050 基极 (B)
NPN 发射极 (E) → GND
NPN 集电极 (C) → 有源蜂鸣器 → VCC (+5V)
P2.7 = 1 → NPN 导通 → 蜂鸣器响
```

**基极电流计算：**
- P2.7 为准双向口，输出高电平时内部弱上拉等效 ~50kΩ
- 基极电流路径：VCC → 50kΩ(内部上拉) → P2.7 引脚 → 500Ω → 基极 → 0.7V(V_BE) → GND
- I_B ≈ (5V - 0.7V) / (50kΩ + 500Ω) ≈ 4.3V / 50.5kΩ ≈ **85μA**

**集电极电流计算：**
- 8050 NPN 三极管 hFE ≈ 100-200
- I_C = hFE × I_B ≈ 100 × 85μA = **8.5mA** (最小值)
- 典型有源蜂鸣器 (5V) 工作电流：**30-100mA**

**判定：** ⚠️ **蜂鸣器驱动电流可能不足。**

| 参数 | 当前值 | 需求值 | 判定 |
|:---|:---:|:---:|:---:|
| 基极电流 I_B | 85μA | >500μA (保证饱和) | ❌ 不足 |
| 集电极电流 I_C | ~8.5mA | 30-100mA | ❌ 不足 |
| 8050 最大 I_C | 500mA | — | ✅ 余量充足 |

**问题根源：** P2.7 准双向口高电平输出驱动能力太弱 (等效 50kΩ 上拉)，无法为 NPN 基极提供足够电流。

**修正建议：**

**方案 A（推荐）：增加基极上拉电阻**
```
在原 500Ω 基极电阻基础上，增加 P2.7 到 VCC 的 1kΩ 上拉电阻。
```
- 等效上拉：50kΩ // 1kΩ ≈ 980Ω
- I_B ≈ (5V - 0.7V) / (980Ω + 500Ω) ≈ 4.3V / 1480Ω ≈ **2.9mA**
- I_C = 100 × 2.9mA = 290mA，足够驱动蜂鸣器
- 增加 1 个 1kΩ 电阻

**方案 B：改为 PNP 驱动**
```
P2.7 → 1kΩ → PNP 基极 (S8550)
PNP 发射极 → VCC
PNP 集电极 → 蜂鸣器 → GND
P2.7 = 0 → PNP 导通 → 蜂鸣器响
```
- 逻辑反转：P2.7=0 蜂鸣器响
- 需要修改代码中蜂鸣器控制逻辑
- 不需要续流二极管 (有源蜂鸣器内部有振荡电路)

**方案 C（仅 Proteus 仿真）：保持现有设计**
- Proteus 中端口模型可能理想化，仿真中蜂鸣器可能正常
- 标注"仿真专用，实际硬件需增加基极驱动能力"

### 3.3 蜂鸣器续流二极管

**判定：** ✅ **不需要续流二极管。** 有源蜂鸣器内部集成了振荡电路和驱动三极管，不是感性负载 (电磁式蜂鸣器)，不需要续流二极管。若使用电磁式无源蜂鸣器，则需在蜂鸣器两端并联 1N4148 续流二极管 (阴极接 VCC)。

### 4.2 按键消抖验证

**判定：** ✅ **消抖机制正确。**

| 参数 | 值 | 验证 |
|:---|:---|:---|
| 消抖采样次数 | DEBOUNCE_CNT = 3 | ✅ 连续 3 次确认 |
| 扫描周期 | keyTicks >= 5 → 10ms | ✅ 每次 2ms 中断 × 5 |
| 总消抖时间 | 3 × 10ms = 30ms | ✅ 在 20-50ms 标准范围内 |
| 边沿检测 | prev==0 && stable!=0 (上升沿) | ✅ 防长按重复触发 (Bug #3) |

### 4.3 按键映射矛盾验证

**问题：** 用户声称 P3.6=紧急键, P3.7=夜间键，但硬件设计文档和 deep_check 分析均确认 P3.0=紧急键, P3.1=夜间键。

**证据链：**
1. 硬件设计文档 (02) 第三节 P3 口引脚分配表明确列出：P3.0=KEY_EMERGENCY, P3.1=KEY_NIGHT
2. P3.6 和 P3.7 在硬件文档中标注为"未使用"
3. 元器件清单共 6 个功能按键，对应 P3.0-P3.5，再无多余按键
4. 代码审查报告确认函数 KeyReadRaw 读取 6 个按键 (P3.0-P3.5)

**判定：** ✅ **硬件设计文档为准，P3.0=紧急键, P3.1=夜间键 是正确的。** 用户的声称 P3.6/P3.7 与硬件文档矛盾，以硬件文档为准。

### 5.1 晶振频率验证

**判定：** ⚠️ **11.0592MHz 选择存疑但可接受。**

| 频率 | 定时器时钟 | 2ms 计数 | 重载值 | 误差 |
|:---|:---|:---|:---|:---|
| 11.0592MHz | 921.6kHz | 1843.2 | 0xF8CD (1843) | 0.01% |
| 12MHz | 1MHz | 2000 | 0xF830 (2000) | 0% |

**分析：**
- 11.0592MHz 传统上用于需要精确波特率的串口通信 (9600/19200 等)
- P-B-5# 项目**无串口通信**需求
- 12MHz 可提供精确的整数定时 (2ms 计数 = 2000，零误差)
- 人机协作反思中，人类开发者称"选择 11.0592MHz 是经验性决策"
- 11.0592MHz 的 0.01% 定时误差可忽略，不影响功能

**结论：** 功能上无影响，但 12MHz 在数学上更优。建议在文档中注明选择原因。

### 5.2 复位电路验证

**判定：** ✅ **复位电路正确。**

- RC 时间常数：τ = 10kΩ × 10μF = 100ms
- 上电时 C3 充电，RST 保持高电平约 100ms，远大于 AT89C52 要求的 2 个机器周期 (24 个振荡周期 ≈ 2.17μs)
- 手动复位按键并联在 C3 两端，按下时 C3 放电、RST 拉高，松开后重新充电复位
- 电路为标准 51 单片机上电复位电路

**注意：** 硬件文档中描述为 "R1(10kΩ) VCC→RST, C3(10μF) RST→GND"，这是标准的高电平复位电路。但文档中写的是"R1: 10kΩ, VCC → RST(9)"和"C3: 10μF 电解, RST(9) → GND"，这与标准电路一致。确认无误。

### 5.3 EA 引脚验证

**判定：** ✅ **EA=1 (接 VCC) 正确。** AT89C52 使用内部 8KB Flash 程序存储器，不使用外部 ROM。

### 6.2 关键器件逐项分析

#### 6.2.1 数码管 7SEG-MPX2-CC

**Proteus 支持：** ✅ 7SEG-MPX2-CC 是 Proteus 原生器件，搜索名称确认。

**电气兼容性：**
- 共阴数码管，段选需高电平点亮，位选需低电平选通
- 段驱动来自 P0 (10kΩ 上拉)，驱动能力不足 (见 1.3)
- 位选驱动来自 P2 (灌电流)，若 P0 上拉改小则驱动能力不足 (见 3.1)

**判定：** ⚠️ **器件本身正确，但驱动电路需补强。** 建议增加 ULN2003A 作为位选驱动，并将 P0 上拉改为 470Ω 或增加 74HC245。

#### 6.2.2 OLED SSD1306 I2C

**Proteus 支持：** ⚠️ **Proteus 8 无原生 SSD1306 模型。**

Proteus 8 支持的 OLED/显示器模型包括：
- LCD 字符型 (LM016L, LM044L 等)
- 图形 LCD (PG12864F 等)
- TFT 显示屏 (部分型号)

SSD1306 I2C OLED **不在 Proteus 8 标准器件库中**。可能的替代方案：
- 使用虚拟终端 (VIRTUAL TERMINAL) 显示调试信息
- 使用 I2C 调试器 (I2C DEBUGGER) 监控 I2C 总线数据
- 使用 PG12864F (并行接口) 替代，但需修改驱动代码和接线
- 使用第三方 SSD1306 Proteus 模型库 (需额外下载)

**电气兼容性：**
- I2C 总线缺少 4.7kΩ 上拉电阻 (见 2.2)
- I2C 地址 0x78 (7-bit 0x3C) 正确
- 供电 3.3V/5V 兼容 (SSD1306 模块通常板载 LDO)

**判定：** ⚠️ **Proteus 仿真中 OLED 可能无法正常工作，且 I2C 缺少上拉电阻。** 建议：
1. 寻找第三方 SSD1306 Proteus 库
2. 或改用 PG12864F 图形 LCD + 并行接口
3. 补充 I2C 上拉电阻 (4.7kΩ × 2)
4. 在仿真受限情况下，可用 I2C DEBUGGER 验证通信数据正确性

#### 6.2.3 LED (红/黄/绿)

**Proteus 支持：** ✅ LED-RED, LED-YELLOW, LED-GREEN 均为原生器件。

**电气兼容性：** ❌ 拉电流驱动不可行 (见 2.1)。必须改为灌电流驱动或增加 PNP 驱动。

**判定：** ❌ **LED 驱动方式错误，需修改电路和代码。**

#### 6.2.4 蜂鸣器 + NPN 驱动

**Proteus 支持：** ✅ BUZZER (DC/Active) 和 NPN 均为原生器件。

**电气兼容性：** ⚠️ P2.7 准双向口高电平驱动能力不足，NPN 基极电流仅 85μA (见 3.2)。

**判定：** ⚠️ **驱动能力不足，需增加基极上拉电阻或改用 PNP 驱动。**

#### 6.2.5 按键

**Proteus 支持：** ✅ BUTTON 为原生器件。

**电气兼容性：** ✅ 10kΩ 上拉 + 按键接地，标准连接方式。

**判定：** ✅ **按键电路正确。**

## 八、Proteus 仿真 vs 实际硬件差异总结

**关键警告：** 以下问题在 Proteus 仿真中可能**不表现**，但在实际硬件上会**导致功能失效**：

| 问题 | Proteus 仿真 | 实际硬件 | 原因 |
|:---|:---:|:---:|:---|
| P0 10kΩ 上拉驱动数码管 | 可能正常 | ❌ 极暗/不亮 | Proteus 端口模型理想化 |
| P1 拉电流驱动 LED | 可能正常 | ❌ 不亮 | Proteus 不模拟准双向口弱上拉 |
| P2.7 驱动蜂鸣器 | 可能正常 | ❌ 不响/声小 | 同上 |
| I2C 无上拉电阻 | 可能间歇正常 | ❌ 通信失败 | Proteus 不模拟总线电容和上升时间 |
| 位选无驱动 | 可能正常 | ❌ 数码管暗/损坏 | Proteus 不模拟过流和驱动能力 |
| OLED SSD1306 | 无模型 | N/A | 需第三方库 |

**结论：** 当前硬件设计在 Proteus 仿真中基本可运行 (因仿真模型理想化)，但**不能直接用于实际硬件制作**。如需制作实物，必须按本报告建议逐一修正驱动电路。

## 九、完整的修正后电路建议

### 9.1 修正后的 P0 口 (段码驱动)

```
方案 A (推荐): 增加 74HC245 驱动
P0.0-P0.7 → 74HC245 A1-A8 (输入)
74HC245 B1-B8 → 220Ω × 8 → 数码管 a-g,dp 段

```

### 9.2 修正后的 P1 口 (LED 驱动 + I2C)

```
LED 驱动 (灌电流模式):
VCC → LED 阳极 → LED 阴极 → 220Ω → P1.0-P1.5
P1.x = 0 → LED 亮

I2C 上拉:
P1.6 (SCL) → 4.7kΩ → VCC
P1.7 (SDA) → 4.7kΩ → VCC
```

### 9.3 修正后的 P2 口 (位选 + 蜂鸣器)

P2.0 → ULN2003A 1B → 1C → NS 十位共阴端
P2.1 → ULN2003A 2B → 2C → NS 个位共阴端
P2.2 → ULN2003A 3B → 3C → EW 十位共阴端
P2.3 → ULN2003A 4B → 4C → EW 个位共阴端

蜂鸣器驱动 (增加基极上拉):
P2.7 → 500Ω → NPN 8050 基极

NPN 集电极 → 蜂鸣器 → VCC
