## 二、关键逻辑审查

### 2.1 状态机逻辑

**正常模式相位流转：**
```
NS_GREEN → NS_YELLOW → EW_GREEN → EW_YELLOW → (循环)
```
- NS_GREEN 进入时调用 CalcGreenTime 动态计算绿灯时长 ✅
- NS_YELLOW 固定 3 秒 ✅
- EW_GREEN 进入时调用 CalcGreenTime ✅
- EW_YELLOW 固定 3 秒 ✅

**紧急模式：**
- 优先级最高，HandleKeys 中紧急键处理后直接 return，跳过夜间和行人处理 ✅
- 退出紧急模式后调用 StartPhase(PHASE_NS_GREEN) 恢复正常 ✅

**夜间模式：**
- 优先级次之，处理后 return，跳过行人处理 ✅
- 退出夜间后同样调用 StartPhase(PHASE_NS_GREEN) 恢复 ✅

### 2.2 车辆检测逻辑

- RegisterVehicle 中累计总车流量 g_nsFlowTotal/g_ewFlowTotal ✅
- 仅当该方向为红灯时才计入等待队列 g_nsVehicleCnt/g_ewVehicleCnt ✅
- 等待队列上限为 9（代码中 g_nsVehicleCnt < 9 限制）✅

### 2.3 智能配时算法

- CalcGreenTime 中 ownCnt 上限 6（防止溢出） ✅
- 基础 12 秒 + 每车 2 秒 ✅
- 车多方向额外 +3 秒 ✅
- 有行人请求 +3 秒 ✅
- 最终限制在 8~25 秒 ✅

### 2.4 按键消抖

- KeyScan 中使用 3 次采样确认（DEBOUNCE_CNT = 3） ✅
- 边沿检测：prev == 0 && cur != 0 ✅
- 长按只触发一次（上升沿识别） ✅

### 2.5 OLED 局部刷新

- 使用缓存变量（g_oledModeCache 等）避免重复刷新 ✅
- 只在值变化时更新对应区域 ✅
