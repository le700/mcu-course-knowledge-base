## Bug #1：数码管显示重影

**现象：** 数码管切换位选时，前一位的段码会短暂显示在当前位上，出现"鬼影"。  
**原因：** 位选切换和段码输出之间没有消隐操作，旧段码残留在 P0 口。  
**解决：** 在 Timer0ISR 中每次切换前先关闭所有位选（P2.0-P2.3=1），再送新段码，最后才开位选。  
**状态：** ✅ 已解决

```c
/* 消隐处理 */
DIG_NS_TENS = 1;  /* 全关 */
DIG_NS_ONES = 1;
DIG_EW_TENS = 1;
DIG_EW_ONES = 1;
P0 = 0x00;        /* 清段码 */

P0 = g_dispBuf[g_digitIndex];  /* 送新段码 */
switch (g_digitIndex)           /* 开位选 */
...
```

## Bug #2：P0 口驱动能力不足

**现象：** 数码管亮度暗或不亮。  
**原因：** P0 口是开漏输出，没有内部上拉，无法输出高电平驱动 LED。  
**解决：** 在 P0 口外部接 8×10kΩ 上拉电阻到 VCC。  
**状态：** ✅ 已解决

---

## Bug #3：按键长按重复触发

**现象：** 长按紧急键时，紧急模式反复开关。  
**原因：** 按键扫描只检测电平，没有边沿检测，长按期间每次扫描都触发。  
**解决：** 使用上升沿检测（prev==0 && cur!=0），只在按下瞬间触发一次。  
**状态：** ✅ 已解决

---

## Bug #4：夜间模式退出后状态残留

**现象：** 退出夜间模式后，黄灯闪烁标志 g_flashOn 可能为 0，导致黄灯不亮。  
**原因：** 退出夜间时没有重置闪烁状态。  
**解决：** 退出夜间模式时将 g_flashOn 置 1，g_flashTicks 清零，并调用 StartPhase 恢复正常。  
**状态：** ✅ 已解决

---

## Bug #5：OLED 全屏刷新导致闪屏

**现象：** OLED 频繁调用 OLED_Clear() 导致画面闪烁。  
**原因：** 每次更新都清屏再重绘，128×64 的全屏写入耗时约 40ms，造成可见闪烁。  
**解决：** 改用局部刷新 + 缓存比较，只更新变化的区域，避免全屏重绘。  
**状态：** ✅ 已解决

---

## Bug #6：数码管共阴/共阳选错

**现象：** 数码管显示的数字笔画错乱。  
**原因：** Proteus 中选用了共阳数码管 7SEG-MPX2-CA，但代码按共阴输出。  
**解决：** 将 Proteus 元件改为 7SEG-MPX2-CC（共阴），或调整段码表。  
**状态：** ✅ 已解决

## Bug #7：紧急模式方向选择不明确

**现象：** 进入紧急模式时，优先方向选择逻辑不清晰，有时选择错误方向。  
**原因：** 方向选择条件不够完善，没有考虑所有情况。  
**解决：** 增加 SelectEmergencyDirection 函数，综合考虑红外检测方向、等待车辆数和当前通行方向三个因素。  
**状态：** ✅ 已解决

---

## Bug #8：行人请求无效

**现象：** 按下行人键后，系统没有缩短对向绿灯时间。  
**原因：** HandleKeys 中行人处理在紧急/夜间检查之后，但缺少对当前相位的判断。  
**解决：** 在行人处理中增加相位判断：对向绿灯时缩短至 REQUEST_SHORTEN_SEC，本向绿灯时延长至 PED_EXTEND_SEC。  
**状态：** ✅ 已解决

---

## Bug #9：编译警告 L16: UNCALLED SEGMENT

**现象：** 编译时出现 WARNING L16: UNCALLED SEGMENT, IGNORED FOR OVERLAY PROCESS。  
**原因：** 定义了 FlashToggle 函数但从未调用。  
**解决：** 删除未使用的函数。  
**状态：** ✅ 已解决

---

## Bug #10：OLED I2C 通信时序不稳

**现象：** OLED 偶尔显示乱码或不响应。  
**原因：** I2C 延时函数精度不够，51 单片机的 NOP 延时在不同编译器优化下可能不稳定。  
**解决：** 使用 4 个 NOP 作为 I2cDelay，保证最小延时足够。  
**状态：** ✅ 已解决
