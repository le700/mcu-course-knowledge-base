
**配置项验证**:
| 配置 | 代码 | 说明 | 结果 |
|------|------|------|------|
| SCON | 0x50 | 方式1, 8位UART, 允许接收 | 正确 |
| S1ST2 | AUXR \|= 0x01 | Timer2作为波特率发生器 | 正确 |
| T2x12 | AUXR \|= 0x10 | 1T模式 | 正确 |
| T2R | AUXR \|= 0x04 | 启动Timer2 | 正确 |
| T2H | 0xFD | 重载值高字节 | 正确 |
| T2L | 0xC0 | 重载值低字节 | 正确 |
| ES | 1 | 允许串口中断 | 正确 |
| EA | 1 | 开总中断 | 正确 |

**瑕疵**: Timer2初始化顺序问题。

```c
void uart_init(void)
{
    SCON = 0x50;
    AUXR |= 0x01;    // S1ST2=1
    AUXR |= 0x04;    // T2R=1 ← 在此处启动Timer2
    T2L = 0xC0;      // ← 之后才设置重载值
    T2H = 0xFD;      // ← 之后才设置重载值
    AUXR |= 0x10;    // T2x12=1 ← 1T模式在启动之后才设置
    ES = 1;
    EA = 1;
}
```

**问题**: `T2R=1` 在第17行先启动了Timer2，但 `T2L`/`T2H` 在第18-19行才设置，`T2x12=1` 在第20行才设置。这导致Timer2以默认值（或不确定值）和默认12T模式短暂运行，直到配置完成。虽然时间极短（几个机器周期），影响可忽略，但**不符合最佳实践**。

**建议修复**:
```c
void uart_init(void)
{
    SCON = 0x50;
    AUXR |= 0x01;    // S1ST2=1: Timer2作为波特率发生器
    T2L = 0xC0;      // 先设置重载值
    T2H = 0xFD;
    AUXR |= 0x10;    // 再设置1T模式
    AUXR |= 0x04;    // 最后启动Timer2
    ES = 1;
    EA = 1;
}
```

**结论**: 波特率计算结果正确，配置参数正确，但初始化顺序有瑕疵。**通过**。

### 12. 数码管动态扫描消影处理 -- 通过

**文件**: `/workspace/P-A-3工程/common/display.c` (第93-115行)

**消影流程**:

```c
void display_scan_isr(void)
{
    P0 = 0xFF;                       // ① 先关闭段选（全灭）
    Select_HC573(7);                 // ② 锁存段选
    Select_HC573(0);                 // ③ 关闭通道

    P0 = ~weixuan_table[scan_index]; // ④ 设置新位选
    Select_HC573(6);                 // ⑤ 锁存位选
    Select_HC573(0);                 // ⑥ 关闭通道

    P0 = disp_buf[scan_index];       // ⑦ 输出新段码
    Select_HC573(7);                 // ⑧ 锁存段选
    Select_HC573(0);                 // ⑨ 关闭通道

    scan_index++;                    // ⑩ 移到下一位
}
```

**消影原理**: 在切换位选之前，先将段选全部关闭（P0=0xFF），这样可以防止新位选位置短暂显示旧位选位置的段码，消除"鬼影"/"重影"现象。

**扫描频率**: Timer0每2ms中断一次，8位完整扫描周期 = 16ms，刷新率约62.5Hz，远高于人眼闪烁感知阈值（约50Hz），视觉稳定。

**结论**: 消影处理正确，扫描频率合理。**通过**。

### 14. PCF8591 I2C时序 -- 通过（有瑕疵）

**文件**: `/workspace/P-A-3工程/common/pcf8591.c` (第8-112行)

**I2C起始信号**:
```
SDA=1, SCL=1 → SDA=0 → SCL=0  ✓
```

**I2C停止信号**:
```
SDA=0, SCL=1 → SDA=1  ✓
```

**I2C发送字节**:
- 8位数据从MSB开始发送
- 每位: SCL拉高→延时→SCL拉低→延时
- 发送完成后释放SDA，产生第9个时钟用于ACK检测

**I2C读取字节** (`I2C_ReadByte`):
- 释放SDA，8位数据从MSB读取
- 每位: SCL拉高→延时→读SDA→SCL拉低→延时

**PCF8591读取流程**:
```
Start → 0x90(写地址) → 0x43(控制字:通道3) → Restart → 0x91(读地址) → 读ADC → Stop
```

**瑕疵**: 缺少NACK时钟脉冲。

在 `I2C_ReadByte()` 中，读取完8位数据后，函数直接返回，**没有产生第9个时钟脉冲发送NACK信号**。紧接着调用 `I2C_Stop()` 直接产生停止条件。

标准I2C协议要求在读取最后一个字节后，主设备应：
1. 在第9个时钟周期保持SDA=1（NACK）
2. 然后产生停止条件

代码跳过了第9个时钟，直接进入停止条件。虽然**PCF8591在实际使用中通常能容忍这种简化**，但这不符合I2C标准的严格时序要求。

**建议修复**（在 `I2C_ReadByte` 末尾添加NACK时钟）:
```c
static u8 I2C_ReadByte(void)
{
    u8 i, dat = 0;
    SDA = 1;
    for (i = 0; i < 8; i++) {
        dat <<= 1;
        SCL = 1;
        delay_us(5);
        dat |= SDA;
        SCL = 0;
        delay_us(2);
    }
    /* 发送NACK */
    SDA = 1;
    SCL = 1;
    delay_us(5);
    SCL = 0;
    return dat;
}
```

**结论**: 核心时序正确，功能可用，但缺少NACK时钟脉冲。**通过**。

### 问题A: DS18B20负温度处理缺失

**位置**: `pa3_main/src/main.c` 第215行  
**代码**: `temp_val = (u16)((u32)raw * 625 / 100);`

当 `raw` 为负数（即温度低于0°C，MSB=1）时，`(u32)raw` 会被解释为大的正数（如 -1 → 0xFFFF → 65535），导致温度计算错误。在室内温度环境下（>0°C）不会触发，但**缺少负温度保护**。

**建议修复**:
```c
if (raw & 0x8000) {
    /* 负温度 */
    raw = (~raw) + 1;
    temp_val = (u16)((u32)raw * 625 / 100);
    /* 显示负号 */
} else {
    temp_val = (u16)((u32)raw * 625 / 100);
}
```

### 问题B: `delay_us` 函数注释不准确

**位置**: `common/delay.c` 第7-8行  
**注释**: "约 1us 延时"  
**实际**: 每次调用 `delay_us(1)` 实际延时约 3-4us（两个 `_nop_()` + 循环开销约 3-4个机器周期，11.0592MHz下约 3.3-4.3us）。对I2C和1-Wire协议来说，时序窗口足够宽，不影响功能，但注释有误导性。

### 问题C: `ds18b20_read_temp()` 函数未被使用

**位置**: `common/ds18b20.c` 第94-121行

`ds18b20_read_temp()` 函数实现了完整的DS18B20温度读取流程，但在 `main.c` 的 `Update_Display()` 中（第193-208行）又写了一份**内联的重复代码**。这导致：
1. 代码冗余
2. `ds18b20_read_temp()` 函数成为死代码

**建议**: 在 `Update_Display()` 中直接调用 `ds18b20_read_temp()` 替代内联代码。

### 问题D: 段码表注释中位映射描述错误

**位置**: `common/display.c` 第25行  
**注释**: "bit7=a, bit6=b, bit5=c, bit4=d, bit3=e, bit2=f, bit1=g, bit0=dp"  
**实际**: 通过段码表值验证，真实映射为 "bit7=dp, bit6=g, bit5=f, bit4=e, bit3=d, bit2=c, bit1=b, bit0=a"（标准共阴映射）。虽然 `& 0x7F` 操作（清除bit7）恰好用于点亮小数点，功能正确，但注释错误。
