一、引言

1.1 项目背景

本项目为 51 单片机课程综合实践项目，旨在将计算机基础知识、汇编语言 / C 语言程序设计、接口电路设计等理论知识与工程实际相结合，完成一个具备实际工程意义的炉温控制系统，培养学生的创新思维与独立分析、解决问题的能力。

1.2 项目目标

实现炉温的精确采集、数字滤波、PID 调节与 PWM 闭环控制。

完成温度超限报警、LCD 实时显示、历史温度数据记录等功能。

锻炼学生系统设计、电路调试与软件编程的工程实践能力。

1.3 文档范围

本文档适用于炉温控制系统的需求分析、方案设计、开发实现、测试验证与验收全流程，面向项目开发成员、指导教师及验收评审人员。

三、详细功能需求

3.1 温度实时采集

输入：热电偶 / 热电阻传感器输出的模拟电压信号。

处理：信号调理电路对模拟信号进行放大、偏置处理后，送入 ADC0809 转换为 8 位数字量。

输出：数字温度数据，供后续滤波与控制模块使用。

3.2 数字滤波处理

输入：ADC 采集的原始温度数据。

处理：采用中位值滤波或滑动平均滤波算法，剔除异常数据，平滑温度曲线。

输出：滤波后的稳定温度数据。

3.3 PID 控制算法实现

输入：目标温度值、滤波后的实际温度值。

处理：

计算温度偏差：e(t) = 目标温度 - 实际温度

按 PID 公式计算控制量：u(t) = Kp*e(t) + Ki*∫e(t)dt + Kd*de(t)/dt

对控制量进行限幅处理，避免输出超限。

输出：PID 控制量，用于生成 PWM 信号。

3.4 加热器 PWM 控制

输入：PID 输出的控制量。

处理：根据控制量占空比生成 PWM 波，驱动加热器功率管。

输出：PWM 信号，调节加热器加热功率，实现温度闭环调节。

3.5 温度超限报警

输入：滤波后的实际温度值、用户设定的温度上下限。

处理：

实时判断温度是否超出阈值。

若超限，触发蜂鸣器 + LED 声光报警。

温度恢复正常范围后，自动解除报警。

输出：声光报警信号。

3.6 LCD 温度显示

输入：目标温度、实际温度、报警状态。

处理：

格式化温度数据为字符串。

驱动 LCD1602 显示当前温度、目标温度、报警状态等信息。

输出：清晰直观的温度显示界面。

3.7 温度历史数据记录

输入：定时采集的滤波后温度数据。

处理：

按设定时间间隔（如 1min）将温度数据写入 EEPROM。

记录数据包含时间戳（相对时间）与温度值。

采用循环覆盖策略，满后自动覆盖旧数据。

输出：EEPROM 中存储的历史温度记录。

4.2 硬件设计要求

引脚分配：明确 51 单片机 I/O 口与 ADC0809、LCD1602、EEPROM、PWM 输出、报警引脚的连接关系。

抗干扰设计：模拟信号与数字信号布线分离，避免电磁干扰。

功率匹配：加热器驱动电路需满足加热器功率与电压要求，确保安全可靠。

五、软件需求

5.1 开发环境

编程语言：C 语言（兼容 51 单片机汇编语言）

开发工具：Keil C51 编译器、STC-ISP 下载工具

仿真工具：Proteus 电路仿真软件

5.2 软件架构

主程序循环：调度各功能模块，实现 “温度采集→滤波→PID 计算→PWM 输出→显示→存储” 的闭环流程。

中断服务程序：

定时器中断：用于 PWM 生成、定时采集与数据存储。

外部中断：用于报警复位或手动干预。

模块分层：

底层驱动：ADC、LCD、EEPROM、PWM 的硬件驱动。

中间算法：数字滤波、PID 控制算法。

上层应用：显示逻辑、报警逻辑、数据存储逻辑。

5.3 程序流程

系统初始化：I/O 口、定时器、ADC、LCD、EEPROM 等模块初始化。

主循环：

温度采集与滤波

PID 控制量计算

PWM 信号输出

LCD 显示更新

温度超限判断与报警

定时数据存储

中断处理：响应定时器 / 外部中断，执行定时任务或手动干预。
